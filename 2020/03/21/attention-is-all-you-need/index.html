Attention Is All You Need

 
  • Paper
  • Mar 21, 2020

這篇文章想要簡介一下被廣泛使用的Transformer是什麼。

簡介

在以前處理有關sequence的問題，像是文字、語音和影像等，所使用的model不是CNN就是RNN，然而它們各自有一些缺點，像是CNN必須要疊的很深才可以看到比較長範圍的資訊，RNN比較不能應付較長的序列，並且必須sequential的做訓練，不能夠被平行化，而這篇paper提出了另一個一樣能套用在sequence上的model，稱之為Transformer。

方法

底下的圖是paper當中所提出的Transformer的架構，跟一般做language translation的架構蠻像的，由encoder和decoder組成，但可以看到其中有一個比較特別的layer：Multi-Head Attention。

Scaled Dot-Product Attention

在講Multi-Head Attention Layer之前，得先要講一下Scaled Dot-Product Attention，因為Multi-Head Attention Layer其實是由Scaled Dot-Product Attention所組成的，而下圖為Scaled Dot-Product Attention的架構。

假設我們現在有一個sequence input ，其長度為，而每一個元是一個維度的向量，可以想像成是我們已經把一個輸入進來的句子轉換成了維的word embeddings

而圖中的、、，分別代表的是Query、Key、Value，是由不同的weight各自乘上輸入所得到的

Scaled Dot-Product Attention做的事情就是，對每個時間點，用其Query對所有時間點的Key做內積，scale以後過得到attention weight，再與Value做weighted sum。假設我們想要得到的輸出，其流程大概會是底下這樣

  1. 取出的query

  2. 對所有時間點的key做內積，得到原始的attention weight

  3. 對原始的attention weight除上維度開根號，以避免原始值太大，亦即

  4. 將通過，得到最終的attention weight

  5. 利用attention weight對value做weighted sum，得到時間點的輸出

其他時間點的輸出都是利用相同的方式得到的，我們也可以從中發現到，其實產生某個特定時間點的輸出並沒有與其他時間點有關聯，我們可以簡單地用矩陣相乘的方式一次算出所有的輸出

至此我們有了可以輸入一個序列，得到與相同長度輸出的Scaled Dot-Product Attention layer。

Multi-Head Attention Layer

而在Transformer架構中的Multi-Head Attention其實就是把輸入的、、過linear transform、把多個Scaled Dot-Product Attention平行的疊在一起、將每個輸出都接起來後再過一個linear transform

在paper裡面，將的個數設為、，其中的是Multi-Head Attention輸出的維度。

Add & Norm

這一層做的事情是加一個residual connection，並做Layer Normalization。

Positional Encoding

在上面的方法中，在產生特定時間點輸出的時候，並不會對比較近或比較遠的輸入而有不一樣的操作，使得在產生的時候，model並不知道是比較近而是比較遠的，因此在paper裡對輸入都加入了positional encoding，好讓model可以看輸入就知道這個輸入是哪一個時間點的輸入。
